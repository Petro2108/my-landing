## 1. ЦЕЛЬ И АУДИТОРИЯ

### Цель сайта

**Единственная цель:** Побудить владельца бизнеса оставить заявку — через форму, Calendly, email или Telegram.

Никаких дополнительных целей. Один CTA на всей странице, повторяющийся 3 раза.

### Целевая аудитория

**Первичная:** Владелец малого и среднего бизнеса (5–50 сотрудников).
- Слышал про ИИ и цифровизацию, хочет "этого", но не понимает как
- Устал от найма нескольких исполнителей под разные задачи
- Готов платить за результат, а не за часы
- Принимает решения эмоционально, обосновывает логически
- 63–68% зайдут с мобильного

**Вторичная:** Стартап-фаундер на стадии MVP, которому нужен быстрый запуск продукта.

**Антиаудитория (кого не нужно привлекать):**
- Другие разработчики
- Те, кто ищет дешевле $X (ниже минимального порога)
- Enterprise с тендерными процессами

### Уникальное позиционирование

> Один специалист — полный цикл: сайт, приложение, ИИ-автоматизация.
> Без агентства, без менеджеров, без потери смысла при передаче задачи.

**Главный оффер (H1):**
> «Строю сайты, приложения и ИИ-агентов, которые работают на ваш бизнес — пока вы занимаетесь главным»

*(Альтернативы в разделе 3.1)*

## 2. ДИЗАЙН-СИСТЕМА

### 2.1 Настроение и эстетика

**Ключевое слово:** Умный тёмный минимализм с живым дыханием.

Сайт должен транслировать:
- Техническую глубину (ты разработчик, а не дизайнер шаблонов)
- Надёжность (деньги клиента в безопасности)
- Современность (aurora gradients, glassmorphism — ты в тренде)
- Человечность (не корпорация, а живой специалист)

**Референсы:** Vercel.com, Linear.app, Raycast.com — тот же класс читаемости и премиальности, но с личным характером.

**Запрещено:**
- Стоковые фотографии людей
- Яркий белый фон (#fff) как основной
- Больше 3 цветов в акцентах
- Autoplay видео/аудио
- Анимации, блокирующие контент

## 3. СТРУКТУРА СТРАНИЦЫ

### НАВИГАЦИЯ (sticky header)

**Элементы:**
- Логотип / инициалы слева (текст или SVG-монограмма)
- Единственная кнопка справа: «Обсудить проект» (primary CTA)
- Нет пунктов меню — каждый пункт это выход с лендинга

**Поведение:**
- При скролле вниз: `backdrop-filter: blur(12px)` + `background: rgba(15,23,42,0.8)`
- При скролле вверх: полностью прозрачный
- На мобильном: логотип + кнопка «Написать»

```
[H1 — главный заголовок]
[Лид — 1–2 предложения]
[Два CTA-кнопки]
[Социальное доказательство — под кнопками]
[Визуальный элемент — справа или снизу]
```

H1 (основной вариант):

> **которые работают на ваш бизнес**

*(слово «работают» или «бизнес» — gradient-text violet→cyan)*

Альтернативные заголовки (выбрать один после A/B теста):
> «От идеи до работающего цифрового продукта — без передачи задачи по кругу»

> Помогаю малому и среднему бизнесу цифровизироваться без боли.
> Разрабатываю сайты, веб-приложения и системы автоматизации на ИИ — сам, без посредников, с результатами в числах.

**Кнопки:**
- Primary: `Обсудить проект →` (зелёный #22C55E, magnetic hover)
- Secondary: `Посмотреть работы ↓` (ghost, белый border)

**Социальное доказательство под кнопками:**
```
★★★★★  ·  [XX проектов]  ·  [X лет опыта]  ·  [XX довольных клиентов]
```
*(либо 3–4 логотипа клиентов в grayscale)*

**Визуальный элемент:**
- Вариант A (рекомендован): Spline 3D-объект — абстрактная геометрия или «рабочий стол» с предметами, реагирующий на движение мыши
- Вариант B (запасной): Анимированный GIF/видео с демо одного из проектов в рамке устройства
- Вариант C (минимальный): Анимированный code-блок с реальным кодом проекта (typewriter эффект)

**Технические требования:**
- LCP элемент — `<img>` с `preload`, не `background-image`
- Вся секция видна без скролла на 1280px и 390px (iPhone 14)
- Минимальная высота hero: 100dvh

### СЕКЦИЯ 2: ПРОБЛЕМА / РАСПОЗНАВАНИЕ БОЛИ

**Цель:** Создать эмпатию. Клиент думает: «Это про меня».

**Заголовок секции:**
> «Знакомая ситуация?»

**Контент — 4 карточки-боли (bento grid 2×2):**

Карточка 1:
> 😤 **«Уже третий исполнитель — и снова объясняю с нуля»**
> Дизайнер, разработчик, сеошник, ещё кто-то. Каждый знает своё, никто не видит картину целиком.

Карточка 2:
> 📉 **«Сайт есть, но клиентов с него нет»**
> Деньги потрачены, сайт красивый. Только заявок с него — ноль. Потому что красота ≠ конверсия.

Карточка 3:
> 🤖 **«Все говорят про ИИ, но непонятно что с этим делать»**
> Слышал, что можно автоматизировать. Но как именно применить к моему бизнесу — загадка.

Карточка 4:
> ⏰ **«Лиды приходят ночью, а отвечаем утром — они уже ушли»**
> Скорость ответа решает. Конкурент перезвонил через 3 минуты, а вы — через 6 часов.

**Переход к следующей секции:**
> «Если хоть один пункт попал в точку — читайте дальше. Именно для этого я существую.»

### СЕКЦИЯ 6: ПОРТФОЛИО / КЕЙСЫ

**Цель:** Доказать на примерах. Переход от «говорит что хорош» → «я вижу результаты».

**Заголовок:**
> «Результаты, а не картинки»

**Макет: Bento Grid**

Верхний ряд — один большой кейс (занимает 2/3 ширины):
```
[Скриншот/превью]   [Название проекта]
                    [Клиент: тип бизнеса]
                    [Задача: 1 предложение]
                    [Что сделано: 1–2 предложения]
                    [Результат:]
                    ✓ +Х% заявок за первый месяц
```

Нижний ряд — 2–3 карточки меньшего размера с теми же элементами, но компактнее.

**Структура каждого кейса (обязательно):**
1. Ситуация: кто клиент, какая была проблема
2. Решение: что построил (plain language, без жаргона)
3. Результат: конкретные числа (% роста, часы, деньги, скорость)

**Для ИИ-агентов:** обязательно видео-демо или GIF — продукт невидим без демонстрации.

**CTA под портфолио:**
> «Нужен похожий результат? →  Обсудить проект»

3. На надёжность (сделал как обещал, в срок, без доп. счетов)

**Запрещённые отзывы:**
- «Всё понравилось! Рекомендую.» — без имени, без результата
- Анонимные или без компании

### СЕКЦИЯ 8: FAQ / СНЯТИЕ ВОЗРАЖЕНИЙ

**Формат:** Аккордеон (expand/collapse), 7 вопросов

**Вопросы и ответы:**

---
**Сколько стоит?**
> Стоимость зависит от объёма, но ориентиры такие:
> — Сайт (лендинг): от X до X руб.
> — Веб-приложение (MVP): от X руб.
> — ИИ-автоматизация: от X руб.

---
**Как долго делается проект?**
> Лендинг — 2–3 недели. MVP приложения — 4–8 недель. ИИ-агент — 1–2 недели.

---
**Буду ли я владеть кодом и аккаунтами?**
> Да, всегда и полностью. Код, домен, хостинг, соцсети, Analytics — всё передаётся вам.

---
**Нужно ли мне понимать технологии?**
> Нет. Я объясняю всё на человеческом языке: что делаю, зачем, что это даст.

---
**Что если мне не понравится результат?**
> В проект включены 2 раунда правок. Если что-то не так — исправляю.

---
**Вы работали с бизнесами в моей сфере?**
> Работал с [перечислить 3–4 отрасли]. Если вашей нет — это хорошо:
> привнесу лучшие практики из других сфер. Записывайтесь на звонок — расскажу конкретнее.

---
**Что если мне нужно всё сразу — сайт, приложение и ИИ?**
> Отлично — именно для таких задач я и нужен. Не придётся объяснять контекст
> разным людям: вижу картину целиком и строю систему, а не набор несвязанных инструментов.

## 4. ИНТЕРАКТИВНЫЕ ЭЛЕМЕНТЫ И АНИМАЦИИ

### 4.1 Кастомный курсор

```
Дефолтный: маленькая точка (8px) цвета accent-violet
Hover на ссылках/кнопках: кольцо (40px), accent-cyan, scale up
Hover на проектах: текст "Открыть" появляется внутри кольца
Поведение: lerp-сглаживание (коэффициент 0.12–0.15)
Мобильный: отключён
prefers-reduced-motion: отключён
```

### 4.2 Hero анимации (при загрузке)

Последовательность (не одновременно, stagger):
1. `0ms` — fade in фоновый gradient
2. `100ms` — slide up бейдж-тег
3. `300ms` — SplitText анимация H1 (слова появляются снизу вверх, stagger 50ms)
4. `600ms` — fade in лид-текст
5. `800ms` — slide up кнопки
6. `1000ms` — fade in social proof строка
7. `Параллельно` — 3D объект / визуальный элемент (lazy load)

Длительности: 600–800ms, easing: `cubic-bezier(0.16, 1, 0.3, 1)` (expo out)

### 4.3 Scroll-triggered анимации (GSAP ScrollTrigger)

Все секции при появлении в viewport:
```
trigger: start: "top 80%"
from: { opacity: 0, y: 40 }
to: { opacity: 1, y: 0, duration: 0.7, ease: "power2.out" }
stagger: 0.1 (для групп элементов)
```

Bento-сетка проектов: карточки появляются с разным delay (0, 0.1, 0.2с)

Числа-метрики: counter animation (0 → XX) при входе в viewport

### 4.4 Кнопки — Magnetic hover

CTA-кнопки реагируют на приближение курсора:
```javascript
// Кнопка притягивает курсор в радиусе 80px
// При hover: scale(1.05) + тень расширяется
// При click: scale(0.97) + тень схлопывается
```

### 4.5 Карточки проектов

```
Hover state (desktop):
- translateY(-4px)
- box-shadow: 0 20px 40px rgba(99,102,241,0.2)
- border-color: rgba(99,102,241,0.4)
- duration: 200ms ease

Внутри карточки при hover:
- Скриншот: scale(1.03) с overflow:hidden
- Теги стека: fade in снизу
- «Открыть →» ссылка: появляется
```

### 4.6 Процесс-секция

Горизонтальная прокрутка на десктопе (pinned section):
- Секция прикрепляется при входе в viewport
- При скролле вниз — горизонтально прокручиваются шаги
- Каждый шаг появляется с fade+slide при активации
- Активный шаг: номер горит accent-цветом, линия между шагами «рисуется»

### 4.7 Навигация при скролле

```
При скролле вниз > 80px:
  header: backdrop-filter: blur(16px)
          background: rgba(15,23,42,0.85)
          border-bottom: 1px solid rgba(255,255,255,0.08)
          transition: all 200ms

При скролле вверх:
  header: фон прозрачный, border убирается
```

### 4.8 Аккордеон FAQ

```
Открытие: высота разворачивается (max-height: 0 → auto)
  duration: 300ms, easing: ease-out
Иконка: + поворачивается в × (rotate 45deg)
Одновременно открыт только один элемент
```
