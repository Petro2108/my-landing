- Типографика: Syne / Inter / JetBrains Mono

**Интерактив:**
- Кастомный курсор с lerp-сглаживанием (desktop)
- Cursor trail (6 точек, GPU-friendly)
- 3D tilt карточек на mousemove (desktop)
- Magnetic buttons (притяжение к курсору)
- Scroll reveal (IntersectionObserver)
- Counter animation (0 → N+)
- Parallax aurora blobs (desktop)
- FAQ аккордеон (один элемент открыт)
- Мобильное меню (гамбургер → fullscreen overlay)
- Кнопка «Наверх» (появляется после 340px)
- Scroll progress bar (gradient полоска)
- Форма с SVG-анимацией успеха + конфетти
- Floating + shimmer на code window
- Typing-эффект на badge при загрузке
- Hero particles (18 плавающих точек)

**Технические:**
- Responsive: 390px (iPhone 14) ↔ 1440px+
- Без внешних JS-библиотек (чистый HTML/CSS/JS)
- Schema.org Person + Offers (JSON-LD)
- Skip-to-content ссылка
- prefers-reduced-motion поддержка
- Размер файла: ~90 KB, открывается офлайн

### 1.2 Портфолио — 3–4 реальных кейса

Формат каждого кейса (строго):
```
Клиент: [тип бизнеса, без названия если NDA]
```

### 1.4 Медиафайлы

## ФАЗА 4 — ПЕРЕНОС НА NEXT.JS (PRODUCTION СТЕК)

**Цель:** Переписать index.html на Next.js 15 + TypeScript для масштабирования.  
**Срок:** 7–10 дней  
**Приоритет:** 🟢 Желательно (после старта с index.html)  
**Когда делать:** после первых реальных заявок с сайта

### Почему это нужно

| Сейчас (index.html) | После (Next.js) |
|---|---|
| Один файл, всё в нём | Разделение на компоненты |
| Нет CMS для кейсов | MDX файлы → позже Sanity |
| Форма: Formspree/Web3Forms | API route (`/api/contact`) |
| Ручное обновление | Git push → Vercel автодеплой |
| Нет server-side rendering | SSR/SSG, лучший SEO |
| ~90 KB HTML | Code splitting, lazy load |
| Нет TypeScript | Type safety, рефакторинг без страха |

### Структура проекта

```
my-landing/
├── app/
│   ├── layout.tsx          — шрифты, метаданные, Schema.org
│   ├── page.tsx            — главная (все секции)
│   ├── not-found.tsx       — 404 с CTA
│   └── api/
│       └── contact/
│           └── route.ts    — обработка формы
├── components/
│   ├── sections/
│   │   ├── Hero.tsx
│   │   ├── Problem.tsx
│   │   ├── Services.tsx
│   │   ├── Proof.tsx
│   │   ├── Process.tsx
│   │   ├── Portfolio.tsx
│   │   ├── Testimonials.tsx
│   │   ├── FAQ.tsx
│   │   └── CTA.tsx
│   ├── ui/
│   │   ├── Button.tsx
│   │   ├── Card.tsx
│   │   ├── Badge.tsx
│   │   └── Accordion.tsx
│   └── layout/

│       └── Footer.tsx
├── content/
│   └── cases/

    └── animations.ts       — GSAP хелперы
```

### Порядок миграции

1. `npx create-next-app@latest --typescript --tailwind --app`
2. Перенести CSS-переменные дизайн-системы → `globals.css`
3. Создать компоненты секций (копировать HTML, обернуть в JSX)
4. Настроить GSAP + ScrollTrigger через `useEffect`
5. Перенести кейсы в MDX
6. Добавить API route для формы
7. Деплой на Vercel (`vercel deploy`)

## ФАЗА 6 — РОСТ И АВТОМАТИЗАЦИЯ

**Срок:** 3–6 месяцев после запуска  
**Приоритет:** 🔵 Долгосрочно

### 6.1 Блог / Статьи

SEO-трафик через полезный контент:

Темы для статей:
- «Как выбрать технологию для своего сайта в 2026»
- «Что такое ИИ-агент и зачем он вашему бизнесу»
- «За сколько реально сделать MVP: честный разбор»
- «5 ошибок при заказе сайта»

Реализация: MDX страницы `/blog/[slug]` или внешний Substack/Telegraf.

### 6.2 A/B тестирование

Тестировать поочерёдно:
1. Варианты H1 (3 заготовленных в брифе)
2. Цвет CTA-кнопки (зелёный vs. фиолетовый)
3. Форма vs. Calendly в CTA-секции
4. Позиция секции «Процесс» (до/после портфолио)

Инструменты: Vercel Edge Config + A/B middleware (без внешних платных сервисов).

### 6.3 Email-маркетинг

После заявки → автоматическая серия:
```
День 0: «Получил вашу заявку, вот что дальше...»
День 1: Кейс, релевантный их запросу
День 3: FAQ — закрытие возражений
День 7: Напоминание + другой кейс
```

Инструменты: Resend (бесплатно до 3 000 писем/мес) + React Email.

- `/en/` — английская версия
